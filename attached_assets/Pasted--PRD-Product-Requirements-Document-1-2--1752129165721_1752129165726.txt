## 제품 개요

### 🎯 제품명
**스무고개 금융앱 (Twenty Questions Financial Assistant)**

### 📝 제품 설명
AI 대화형 인터페이스를 통해 시니어 사용자가 금융 서비스를 쉽고 안전하게 이용할 수 있도록 지원하는 혁신적인 금융앱

### 🌟 핵심 가치 제안 (Value Proposition)
- **적시 개입**: 사용자가 막히는 순간을 감지하고 자연스럽게 도움 제공
- **인간적 소통**: 기계가 아닌 사람과 대화하는 듯한 따뜻한 경험
- **점진적 학습**: 개인 수준에 맞는 맞춤형 가이드로 디지털 자신감 구축
- **포용적 금융**: 디지털 소외계층을 위한 진정한 금융 접근성 실현

---

## 문제 정의 및 기회

### 🚨 현재 문제점

#### 시니어 사용자의 디지털 금융 소외
- **통계 데이터**: 60세 이상 인구의 85%가 모바일 뱅킹 첫 시도 후 포기
- **주요 원인**:
  - 복잡한 UI/UX 구조
  - 전문용어 남용
  - 실시간 도움말 부재
  - 실수에 대한 두려움

#### 기존 솔루션의 한계
- **단순 글자 크기 확대**: 근본적 문제 해결 안됨
- **오프라인 교육**: 확장성 부족, 일시적 효과
- **가족 의존**: 독립성 저해, 지속 가능하지 않음

### 🌟 시장 기회
- **시니어 인구 증가**: 2030년 65세 이상 인구 25% 돌파 예상
- **디지털 전환 가속화**: 코로나19로 인한 비대면 서비스 필수화
- **정부 정책 지원**: 디지털 포용 정책 강화
- **기술 성숙도**: AI/ML 기술 발달로 개인화 서비스 구현 가능

- 금융 외 다른 도메인 확장 (의료, 쇼핑 등)

#### 비즈니스 지표

- **거래 성공률**: 95% 이상
- **고객 지원 요청 감소율**: 70% 이상

## 타겟 사용자 및 페르소나

### 👥 주요 타겟 사용자

#### Primary: 디지털 초보 시니어 (65-75세)
- 스마트폰 보유하지만 복잡한 앱 사용 어려움
- 기본적인 금융 거래 필요 (송금, 잔액 조회 등)
- 가족 도움 없이 독립적으로 생활하고 싶어함

#### Secondary: 디지털 경험 있는 시니어 (55-65세)
- 기본적인 스마트폰 사용 가능
- 새로운 기능 학습에 의욕 있음
- 안전하고 편리한 금융 서비스 선호

### 🎭 주요 페르소나

#### 페르소나 1: 김순자 (72세, 주부)
**배경**
- 평생 은행 창구 이용, 디지털 기기 경험 제한적
- 자녀가 설치해준 은행앱 있지만 사용 못함
- 손자에게 용돈 보내는 것이 주요 니즈

**목표**
- 은행 방문 없이 집에서 송금하기
- 가족에게 부담 주지 않고 독립적으로 생활
- 디지털 기기에 대한 두려움 극복

**페인포인트**
- "잘못 누르면 돈이 엉뚱한 곳으로 갈까봐"
- 복잡한 메뉴와 전문용어에 압도됨
- 실수했을 때 해결 방법 모름

**사용 시나리오**
- 매주 목요일 손자 용돈 송금
- 명절 전 자녀들에게 용돈 송금
- 가끔 통장 잔액 확인

#### 페르소나 2: 박영수 (68세, 퇴직자)
**배경**
- 카카오톡, 유튜브 등 기본 앱 사용 가능
- 은행앱 몇 번 시도했지만 복잡해서 포기
- 온라인 쇼핑 경험 있음

**목표**
- 다양한 금융 서비스 활용 (투자, 대출 등)
- 디지털 금융 혜택 누리기
- 새로운 기술 배우며 활발한 노후 생활

**페인포인트**
- 금융 전문용어 이해 어려움
- 보안 절차가 복잡함
- 실시간 도움 받을 곳 없음

**사용 시나리오**
- 월 2-3회 각종 요금 납부
- 주식 투자 정보 확인
- 해외여행 전 환전 서비스

## 핵심 기능 요구사항

### 🤖 AI 대화형 어시스턴트

#### F001: 스무고개 대화 엔진
**기능 설명**: 사용자의 행동과 상황을 분석하여 적절한 질문과 가이드 제공

**상세 요구사항**:
- 사용자 행동 패턴 실시간 분석
- 자연어 처리를 통한 의도 파악
- 개인화된 질문 생성 및 제공
- 감정 상태 인식 및 적절한 톤 조절

**수용 기준**:
- 사용자 의도 파악 정확도 90% 이상
- 응답 속도 2초 이내
- 개인화 적중률 85% 이상

#### F002: 상황 인식 기반 개입
**기능 설명**: 사용자가 막히거나 헤매는 순간을 감지하고 자동으로 도움 제공

**상세 요구사항**:
- 화면 체류시간 분석 (10초 이상 시 개입)
- 반복 행동 패턴 감지
- 오류 발생 시 즉시 개입
- 사용자 동의 하에 화면 안내

**수용 기준**:
- 개입 필요 상황 감지율 95% 이상
- 불필요한 개입 비율 5% 이하
- 사용자 만족도 8/10 이상

### 💬 자연스러운 대화 인터페이스

#### F003: 음성 및 텍스트 지원
**기능 설명**: 사용자가 편한 방식으로 소통할 수 있는 멀티모달 인터페이스

**상세 요구사항**:
- 음성 인식 및 합성 (한국어 특화)
- 텍스트 입력 지원 (큰 글씨, 간단한 키보드)
- 사용자 선호도에 따른 모드 전환
- 배경 소음 필터링

**수용 기준**:
- 음성 인식 정확도 95% 이상
- 음성 합성 자연스러움 점수 4.5/5.0
- 모드 전환 소요시간 1초 이내

#### F004: 개인화된 언어 사용
**기능 설명**: 사용자 수준에 맞는 쉬운 언어와 설명 제공

**상세 요구사항**:
- 금융 전문용어 → 일상 언어 변환
- 사용자 이해도에 따른 설명 깊이 조절
- 지역 방언 및 표현 인식
- 점진적 용어 학습 지원

**수용 기준**:
- 언어 이해도 측정 점수 9/10 이상
- 전문용어 회피율 90% 이상
- 사용자 언어 만족도 9/10 이상

### 🛡️ 안전성 및 보안

#### F005: 다단계 안전 확인
**기능 설명**: 중요한 금융 거래 시 사용자가 이해할 수 있는 방식으로 안전 확인

**상세 요구사항**:
- 거래 내용 음성 및 시각적 재확인
- 단계별 확인 절차 (받는 사람, 금액, 용도)
- 취소 및 수정 기회 제공
- 의심스러운 거래 탐지 및 경고

**수용 기준**:
- 거래 정확도 99.9% 이상
- 사기 거래 탐지율 95% 이상
- 사용자 안전감 점수 9/10 이상

#### F006: 생체인증 간소화
**기능 설명**: 시니어 친화적인 생체인증 시스템

**상세 요구사항**:
- 지문, 얼굴, 음성 인식 지원
- 인증 실패 시 대안 방법 제시
- 사용자 친화적 안내 메시지
- 보안 수준 유지하면서 편의성 증대

**수용 기준**:
- 생체인증 성공률 98% 이상
- 인증 소요시간 3초 이내
- 사용자 편의성 점수 8/10 이상

### 📚 학습 및 성장 지원

## 사용자 여정 및 사용 시나리오

### 🎬 주요 사용 시나리오

#### 시나리오 1: 첫 번째 송금 (김순자 할머니)
```
1. 앱 실행 → "안녕하세요! 오늘 무엇을 도와드릴까요?"
2. 사용자 망설임 감지 → "혹시 송금하시려고 하시나요?"
3. "네, 손자한테 용돈 보내려고요"
4. "좋은 생각이세요! 차근차근 도와드릴게요"
5. 단계별 가이드로 송금 완료
6. "성공하셨어요! 정말 잘하셨네요"
```

#### 시나리오 2: 반복 학습 (박영수 아저씨)
```
1. 두 번째 방문 → "어? 박영수님! 또 뵙네요"
2. "지난번 송금 잘 하셨죠? 오늘은 뭘 해볼까요?"
3. "통장 잔액도 확인해보고 싶어요"
4. "아, 그것도 쉬워요! 보여드릴게요"
5. 새로운 기능 학습 완료
6. "이제 두 가지나 할 줄 아시네요!"
```

### 📱 상세 사용자 플로우

#### 송금 프로세스
1. **진입**: 앱 실행 → 얼굴/지문 인증
2. **의도 파악**: AI가 사용자 행동 분석 → 적절한 질문
3. **정보 수집**: 받는 사람, 금액 등 단계별 입력
4. **안전 확인**: 입력 정보 재확인 → 사용자 동의
5. **실행**: 거래 처리 → 실시간 상태 안내
6. **완료**: 성공 메시지 → 다음 액션 제안

#### 오류 처리 플로우
1. **오류 감지**: 시스템이 문제 상황 인식
2. **즉시 개입**: "잠깐요, 도와드릴게요"
3. **문제 설명**: 쉬운 언어로 상황 설명
4. **해결 제안**: 구체적인 해결 방법 제시
5. **재시도 지원**: 단계별 재진행 가이드
6. **성공 확인**: 문제 해결 후 격려

## 기술적 요구사항

### 🏗️ 시스템 아키텍처

#### 전체 구조
```
[모바일 앱] ↔ [API Gateway] ↔ [마이크로서비스들]
                                ├─ AI 대화 서비스
                                ├─ 사용자 분석 서비스  
                                ├─ 금융 연동 서비스
                                ├─ 보안 서비스
                                └─ 알림 서비스
```

#### AI/ML 스택
- **대화 엔진**: GPT-4 기반 커스텀 모델
